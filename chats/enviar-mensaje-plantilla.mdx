---
title: "Enviar Plantilla de Mensaje"
description: "Envía una plantilla de mensaje de WhatsApp aprobada a un contacto"
api: "POST /chats/enviar/mensaje/plantilla"
---

Envía una plantilla de mensaje de WhatsApp pre-aprobada a un contacto dentro de una conversación existente. El mensaje se registra en la base de datos y se notifica en tiempo real mediante Pusher.

Este endpoint verifica el límite mensual de envío de plantillas definido por el plan del negocio.

<Warning>
  Usa [Enviar Plantilla (Arquitectura)](/chats/enviar-mensaje-plantilla-arquitectura) si la plantilla incluye parámetros dinámicos (variables, encabezados multimedia o botones de respuesta rápida). Este endpoint es para plantillas simples y estáticas.
</Warning>

## Autenticación

Requiere el encabezado `x-api-key` con el App Key del negocio. Consulta [Autenticación](/authentication).

## Cuerpo de la solicitud

<ParamField body="meta_id" type="string" required>
  Número de WhatsApp del contacto en formato internacional (ej. `5215512345678`).
</ParamField>

<ParamField body="contacto_id" type="number" required>
  ID interno del contacto.
</ParamField>

<ParamField body="chat_id" type="number" required>
  ID interno de la conversación.
</ParamField>

<ParamField body="empresa_id" type="number" required>
  ID del negocio que envía la plantilla.
</ParamField>

<ParamField body="plantilla_id" type="number" required>
  ID interno de la plantilla a enviar.
</ParamField>

<ParamField body="user_id" type="number">
  ID del usuario (agente) que realiza el envío. Se usa para atribución del mensaje.
</ParamField>

<ParamField body="user_name" type="string">
  Nombre del usuario que envía la plantilla. Se muestra en la línea de tiempo del chat.
</ParamField>

## Respuesta

<ResponseField name="status" type="string">
  `"success"` cuando la plantilla fue enviada y registrada. `"error"` cuando el proveedor retornó un fallo.
</ResponseField>

<RequestExample>
```bash
curl -X POST https://beta.api-iobot-desarrollo.com/chats/enviar/mensaje/plantilla \
  -H "Content-Type: application/json" \
  -H "x-api-key: TU_APP_KEY" \
  -d '{
    "meta_id": "5215512345678",
    "contacto_id": 101,
    "chat_id": 55,
    "empresa_id": 1,
    "plantilla_id": 7,
    "user_id": 42,
    "user_name": "Carlos"
  }'
```
</RequestExample>

<ResponseExample>
```json 200 Éxito
{
  "status": "success"
}
```

```json 200 Error de envío
{
  "status": "error",
  "error": "Error en el envío del mensaje de plantilla"
}
```

```json 400 Límite mensual alcanzado
{
  "status": "warning",
  "error": "Limite de plantillas alcanzado"
}
```

```json 403 Negocio no encontrado
{
  "estatus": "error",
  "error": "La empresa no existe"
}
```
</ResponseExample>
