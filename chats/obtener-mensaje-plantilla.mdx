---
title: "List Message Templates"
description: "Retrieve the WhatsApp message templates available for a business"
api: "POST /chats/obtener/mensaje/plantilla"
---

Returns all visible, active message templates associated with a business. Templates are fetched from the Meta WABA and enriched with local configuration (visibility, allowed agents, etc.).

If the requesting user has the **agent** role (`role_id: 5`), only templates explicitly assigned to that agent are returned.

## Authentication

Requires [authentication](/authentication) — either an `x-api-key` header (App Key) or a JWT token + `empresa_id` in the request body.

## Request Body

<ParamField body="empresa_id" type="number" required>
  ID of the business whose templates you want to retrieve.
</ParamField>

<ParamField body="usuario_id" type="number" required>
  ID of the user making the request. Used to filter templates when the user is an agent.
</ParamField>

<ParamField body="token" type="string">
  JWT session token. Required when **not** using the App Key (`x-api-key`) authentication method.
</ParamField>

## Response

<ResponseField name="plantillas" type="array">
  List of templates available for the business.

  <Expandable title="Template object">
    <ResponseField name="id" type="number">Internal template ID.</ResponseField>
    <ResponseField name="id_plantilla" type="string">Template name as registered in Meta.</ResponseField>
    <ResponseField name="empresa_id" type="number">Business the template belongs to.</ResponseField>
    <ResponseField name="visible" type="boolean">Whether the template is visible to users.</ResponseField>
    <ResponseField name="agentes_permitidos" type="string">
      Comma-separated list of agent user IDs allowed to use this template. Empty means all agents can use it.
    </ResponseField>
    <ResponseField name="deleted_at" type="string | null">Soft-delete timestamp. `null` for active templates.</ResponseField>
    <ResponseField name="state" type="string">
      Current approval status from Meta. Possible values: `APPROVED`, `PENDING`, `REJECTED`, `no_status`.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash App Key
curl -X POST https://api.iobot.chat/chats/obtener/mensaje/plantilla \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_APP_KEY" \
  -d '{
    "empresa_id": 1,
    "usuario_id": 42
  }'
```

```bash JWT Token
curl -X POST https://api.iobot.chat/chats/obtener/mensaje/plantilla \
  -H "Content-Type: application/json" \
  -d '{
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "empresa_id": 1,
    "usuario_id": 42
  }'
```
</RequestExample>
