---
title: "Campaña masiva desde CSV"
description: "Cómo cargar contactos desde un archivo CSV y enviar una campaña de WhatsApp con plantilla"
---

Este flujo permite enviar una plantilla de WhatsApp a múltiples contactos importados desde un archivo CSV. Consta de dos pasos: cargar el CSV y enviar la campaña.

## Paso 1 — Cargar el CSV

Sube tu archivo CSV al endpoint [`POST /contactos/contacts/upload-csv`](/contactos/subir-csv). El CSV debe tener al menos las columnas `nombre` y `numero`. Puedes agregar columnas adicionales si tu plantilla tiene parámetros dinámicos.

```csv
nombre,numero,ciudad
Juan Pérez,5215512345678,CDMX
María López,5218112345678,Monterrey
```

```bash
curl -X POST https://beta.api-iobot-desarrollo.com/contactos/contacts/upload-csv \
  -H "Authorization: Bearer TU_API_TOKEN" \
  -F "file=@contactos.csv" \
  -F "empresa_id=789" \
  -F "token=TU_USER_TOKEN"
```

La respuesta incluye:

- **`contactos`** — el arreglo de objetos parseados del CSV; lo usarás en el paso 2.
- **`masivos`** — subconjunto de contactos con número válido.
- **`contactsWithNegativeResponses`** — contactos excluidos automáticamente por haber respondido negativamente antes.
- **`errores`** — filas que no pudieron parsearse.

Guarda el arreglo `contactos` y los IDs de contactos devueltos para el siguiente paso.

## Paso 2 — Mapear parámetros (si la plantilla los tiene)

Si tu plantilla tiene variables dinámicas (ej. `{{1}}`, `{{2}}`), decide qué columna del CSV corresponde a cada parámetro.

| Parámetro de plantilla | Columna CSV |
|---|---|
| `{{1}}` | `nombre` |
| `{{2}}` | `ciudad` |

Esto se expresa como el campo `paramMapping`:

```json
{ "1": "nombre", "2": "ciudad" }
```

Si la plantilla no tiene parámetros, omite `paramMapping`.

## Paso 3 — Enviar la campaña

Llama a [`POST /chats/enviar/masivo/plantilla`](/campanias/enviar-masivo-plantilla) con:

- `contacts` — el arreglo `contactos` del paso 1.
- `contacts_id` — los IDs de los contactos.
- `paramMapping` — el mapeo del paso 2.

```bash
curl -X POST https://beta.api-iobot-desarrollo.com/chats/enviar/masivo/plantilla \
  -H "Content-Type: application/json" \
  -H "x-api-key: TU_APP_KEY" \
  -d '{
    "contacts_id": [101, 102],
    "contacts": [
      { "nombre": "Juan Pérez", "numero": "5215512345678", "ciudad": "CDMX" },
      { "nombre": "María López", "numero": "5218112345678", "ciudad": "Monterrey" }
    ],
    "tags": [],
    "plantilla_id": 123,
    "user_id": 456,
    "empresa_id": 789,
    "step": null,
    "nombre": "Campaña Mayo 2026",
    "paramMapping": { "1": "nombre", "2": "ciudad" }
  }'
```

Una respuesta `{ "success": true }` confirma que los mensajes fueron encolados.

## Resumen del flujo

```
CSV file
  └─▶ POST /contactos/contacts/upload-csv
        └─▶ { contactos, contacts_id }
              └─▶ (opcional) mapear columnas CSV → parámetros de plantilla
                    └─▶ POST /chats/enviar/masivo/plantilla
                          └─▶ { success: true }
```
